## 1. CSS 是怎么工作的

### 类比：给积木上色的规则书

CSS 是一本规则书，每条规则说"找到什么样的积木，给它涂成什么颜色、摆成什么形状"。

```
CSS 规则结构：

选择器        声明块
  │          │
  ▼          ▼
h1      { color: red;  font-size: 24px; }
         └────┬────┘  └──────┬───────┘
           属性:值         属性:值
```

**三种引入方式（优先级从低到高）：**

```
外部样式表（推荐）          内部样式           行内样式（优先级最高）
<link rel="stylesheet"    <style>            <p style="color:red">
      href="style.css">     p { color:red }  </p>
                          </style>
```

## 2. 选择器与优先级

### 类比：快递地址的精确程度

选择器就是"地址"，越精确找到的元素越具体：

```
省份（标签选择器）：    p { }           → 所有 <p>
城市（类选择器）：      .card { }       → class="card" 的元素
门牌号（ID选择器）：    #header { }     → id="header" 的元素
精确到人（属性选择器）：[type="email"]  → type 为 email 的 input
```

**优先级权重（记住这张图）：**

```
┌──────────────────────────────────────────────────────┐
│  优先级从高到低                                        │
│                                                      │
│  !important          ████████████████  无限大         │
│  行内 style=""        ████████████     1000           │
│  #id                 ████████         100            │
│  .class / :hover     ████             10             │
│  标签 div / p        ██               1              │
│  *通配符              │               0              │
└──────────────────────────────────────────────────────┘

例：#nav .item a  →  100 + 10 + 1 = 111
   .card h2      →  10 + 1 = 11
   权重高的规则覆盖权重低的
```

## 3. 盒模型

### 类比：快递包裹的四层结构

每个 HTML 元素都是一个快递包裹，从外到内有四层：

```
┌─────────────────────────────────────────────────┐
│                    margin                        │  ← 包裹与包裹之间的距离（透明）
│   ┌─────────────────────────────────────────┐   │
│   │                 border                   │   │  ← 包裹的外壳（可见边框）
│   │   ┌─────────────────────────────────┐   │   │
│   │   │              padding            │   │   │  ← 填充泡沫（内边距）
│   │   │   ┌─────────────────────────┐   │   │   │
│   │   │   │                         │   │   │   │
│   │   │   │        content          │   │   │   │  ← 真正的货物（内容区）
│   │   │   │      width × height     │   │   │   │
│   │   │   │                         │   │   │   │
│   │   │   └─────────────────────────┘   │   │   │
│   │   └─────────────────────────────────┘   │   │
│   └─────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

**box-sizing 的坑（必须理解）：**

```css
/* ❌ 默认 content-box：width 只算内容，加上 padding/border 会撑大 */
.box {
  width: 200px;
  padding: 20px;
  border: 2px solid;
  /* 实际占宽 = 200 + 20×2 + 2×2 = 244px  ← 超出预期！ */
}

/* ✅ border-box（推荐全局设置）：width 包含 padding 和 border */
* { box-sizing: border-box; }
.box {
  width: 200px;
  padding: 20px;
  border: 2px solid;
  /* 实际占宽 = 200px  ← 符合直觉 */
}
```

## 4. Flexbox — 一维布局

### 类比：排队站位

Flexbox 让子元素**排成一排**（或一列），然后控制怎么对齐、怎么分配空间。
就像体育课排队：老师（父容器）决定怎么站，学生（子元素）按指令站位。

```
父容器（flex container）
┌──────────────────────────────────────────────────┐
│  ←────────── 主轴 main axis ──────────────────→  │
│                                                  │
│  ┌────────┐  ┌────────┐  ┌────────┐             │
│  │  子1   │  │  子2   │  │  子3   │             │  ← flex items
│  └────────┘  └────────┘  └────────┘             │
│                                                  │
│  ↕ 交叉轴 cross axis                             │
└──────────────────────────────────────────────────┘
```

**justify-content（主轴对齐）图解：**

```
flex-start:    [1][2][3]                    左对齐
center:              [1][2][3]              居中
flex-end:                    [1][2][3]      右对齐
space-between: [1]        [2]        [3]    两端对齐
space-around:    [1]      [2]      [3]      均匀分布
space-evenly:  [1]    [2]    [3]            等间距
```

**align-items（交叉轴对齐）图解：**

```
stretch（默认）:   flex-start:    center:       flex-end:
┌──┐ ┌──┐ ┌──┐   ┌──┐ ┌─┐ ┌───┐  ┌──┐ ┌─┐ ┌───┐  ┌──┐ ┌─┐ ┌───┐
│  │ │  │ │   │  │  │ │─┘ │   │  │  │ │ │ │   │  │  │ │ │ │   │
│  │ │  │ │   │  └──┘ └─┘ └───┘  │  │ └─┘ │   │  └──┘ └─┘ │   │
│  │ │  │ │   │                   └──┘     └───┘            └───┘
└──┘ └──┘ └───┘
```

```css
.container {
  display: flex;
  flex-direction: row;            /* 横排（默认）| column 竖排 */
  justify-content: space-between; /* 主轴对齐 */
  align-items: center;            /* 交叉轴对齐 */
  flex-wrap: wrap;                /* 超出换行 */
  gap: 16px;                      /* 子元素间距 */
}

.item {
  flex: 1;              /* 等分剩余空间 */
  flex: 0 0 200px;      /* 固定200px，不放大不缩小 */
  align-self: flex-end; /* 单独覆盖交叉轴对齐 */
}
```

**三大常用场景：**

```css
/* 1. 水平垂直居中（最高频！） */
.center { display: flex; justify-content: center; align-items: center; }

/* 2. 导航栏：logo左 + 菜单右 */
.navbar { display: flex; justify-content: space-between; align-items: center; }

/* 3. 响应式卡片 */
.cards { display: flex; flex-wrap: wrap; gap: 16px; }
.card  { flex: 1; min-width: 240px; }
```

## 5. Grid — 二维布局

### 类比：Excel 表格

Flexbox 是排队（一维），Grid 是**表格**（二维），同时控制行和列。
适合整体页面布局，Flexbox 适合组件内部排列。

```
grid-template-columns: 1fr 2fr 1fr

列1(25%)      列2(50%)          列3(25%)
┌──────────┬────────────────────┬──────────┐
│          │                    │          │  行1
├──────────┼────────────────────┼──────────┤
│          │                    │          │  行2
└──────────┴────────────────────┴──────────┘
```

**经典圣杯布局：**

```css
.layout {
  display: grid;
  grid-template-areas:
    "header  header  header"
    "sidebar main    aside"
    "footer  footer  footer";
  grid-template-columns: 200px 1fr 160px;
  grid-template-rows: 60px 1fr 40px;
  min-height: 100vh;
  gap: 0;
}
.header  { grid-area: header;  background: #1a1a2e; }
.sidebar { grid-area: sidebar; background: #f5f5f5; }
.main    { grid-area: main;    padding: 24px; }
.aside   { grid-area: aside;   background: #f5f5f5; }
.footer  { grid-area: footer;  background: #333; }
```

```
┌─────────────────────────────────────────┐
│                 header                   │
├──────────┬──────────────────┬───────────┤
│          │                  │           │
│ sidebar  │      main        │   aside   │
│          │                  │           │
├──────────┴──────────────────┴───────────┤
│                 footer                   │
└─────────────────────────────────────────┘
```

**响应式网格（最实用）：**

```css
/* 自动填充列，每列最小200px，最大等宽 */
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  gap: 16px;
}
/* 屏幕宽时多列，屏幕窄时自动减少列数，无需媒体查询！ */
```

## 6. 响应式设计

### 类比：变形金刚

同一套代码，在手机上是一列，在平板上是两列，在桌面上是三列。

**移动优先（Mobile First）流程：**

```
先写手机样式（最小屏幕，最简单）
        │
        ▼
@media (min-width: 768px)  → 平板增强
        │
        ▼
@media (min-width: 1200px) → 桌面增强
        │
        ▼
不需要写"缩小"逻辑，只写"扩展"逻辑，更简洁
```

```css
/* 手机：单列 */
.cards { display: flex; flex-direction: column; gap: 12px; }

/* 平板：两列 */
@media (min-width: 768px) {
  .cards { flex-direction: row; flex-wrap: wrap; }
  .card  { flex: 0 0 calc(50% - 6px); }
}

/* 桌面：三列 */
@media (min-width: 1200px) {
  .card { flex: 0 0 calc(33.33% - 8px); }
}
```

## 7. 过渡与动画

### 类比：电影的慢动作 vs 定格动画

- `transition`：两个状态之间的**慢动作**（hover 前后）
- `animation`：自定义**定格动画**（可循环、可暂停）

```css
/* transition：hover 时平滑变化 */
.btn {
  background: #42b883;
  transform: translateY(0);
  box-shadow: none;
  transition: all 0.25s ease;   /* 属性 时长 缓动函数 */
}
.btn:hover {
  background: #33a06f;
  transform: translateY(-3px);  /* 上浮 */
  box-shadow: 0 8px 20px rgba(0,0,0,0.15);
}

/* animation：关键帧动画 */
@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(30px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.card {
  animation: fadeInUp 0.4s ease forwards;
}

/* 旋转加载圈 */
@keyframes spin {
  to { transform: rotate(360deg); }
}
.spinner {
  width: 32px; height: 32px;
  border: 3px solid #eee;
  border-top-color: #42b883;
  border-radius: 50%;
  animation: spin 0.8s linear infinite;
}
```

## 8. CSS 变量

### 类比：调色板

把常用颜色、尺寸存在变量里，改一处全局生效。

```css
:root {
  --color-primary:  #42b883;
  --color-text:     #333;
  --color-bg:       #f5f5f5;
  --color-border:   #e0e0e0;
  --radius-sm:      4px;
  --radius-md:      8px;
  --shadow:         0 2px 8px rgba(0,0,0,0.1);
  --transition:     all 0.25s ease;
}

.card {
  background: var(--color-bg);
  border: 1px solid var(--color-border);
  border-radius: var(--radius-md);
  box-shadow: var(--shadow);
  transition: var(--transition);
}

/* 深色模式：只改变量，所有组件自动更新 */
[data-theme="dark"] {
  --color-text:   #eee;
  --color-bg:     #1a1a2e;
  --color-border: #333;
}
```

## 9. 综合练习

### 练习 1：Flex 导航栏（必做）

```
┌──────────────────────────────────────────────────┐
│  🌿 Logo          首页  文章  关于       [登录]   │
└──────────────────────────────────────────────────┘
```
Logo 左对齐，菜单居中，登录按钮右对齐，hover 有过渡效果。

### 练习 2：响应式卡片（必做）

手机1列 → 平板2列 → 桌面3列，卡片 hover 上浮 + 阴影加深。

### 练习 3：圣杯布局（进阶）

用 Grid 实现：顶部导航 + 左侧边栏 + 主内容 + 右侧边栏 + 底部。

### 练习 4：深色模式切换（进阶）

用 CSS 变量 + JS 切换 `data-theme`，实现深色/浅色模式，用 localStorage 记住选择。
